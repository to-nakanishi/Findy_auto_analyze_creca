Credit Card Fraud Detection: 3-Layer Segmentation Strategy  
(リンク先：https://www.kaggle.com/competitions/credit-card-fraud-analysis/overview)

1. プロジェクト概要  
　クレジットカード決済データ（不均衡率0.18%）を用い、不正取引を検知するモデルを構築。  
本プロジェクトでは、単なる精度の追求ではなく、「実ビジネスにおける損失補填額の最大化」  
と「誤検知による顧客離反の抑制」の両立を目的としています。  
　【データセット】  
  - Kaggle Credit Card Fraud Detection  
　- 2013年9月の欧州カード決済データ  
　- 284,807件（不正492件、0.18%）  
　- 匿名化特徴量V1-V28  
  
<br>  

  
2. 独自の分析アプローチ（金額帯別3層戦略）  
　初期手法でベースライン低迷後（Baseline比1％）、EDAを試行錯誤し、不正のパターンが金  
額によって異なることを発見しました。この知見に基づき、一括学習ではなく、金額帯に応じた  
3つのセグメント別モデルを設計しました。  

【層別戦略マトリクス】
| 層 | 区分 | 重視指標 | 戦略的理由 |
| :--- | :--- | :--- | :--- |
| **HIGH** | >$500 | Recall + 金額加重 | 1件あたりの被害額が大きいため、取りこぼしを最小化する |
| **MID** | $100-500 | F1スコア | 検知数と精度のバランスを重視（時間軸との相関が強い領域） |
| **LOW** | <$100 | Precision + F1 | 件数が極めて多いため、誤検知による顧客負担の増加を防止 |

3. 分析のハイライト  
　特徴量の差異: 高額帯（HIGH）では V8 や Amount が重要視される一方、低額帯（LOW）では   
V14 が支配的であることを特定。一括学習では埋もれてしまう「高額不正のシグナル」を捉える    
ことに成功しました。    
　4段階アクションモデル: 予測スコアに基づき、即時の「取引停止」から「モニタリング」や    
「メール確認」まで、実運用を想定した段階的な対応フローを提案。    

  
4. 主な成果  
　ビジネスインパクト: 独自の評価指標（Reduced Cost）において、Baseline比で 36.3%の   
損失削減効果を実証。  
検知額: テストデータにおいて、合計 ＄445,036.71の不正利用を検知。    

  
5. 技術スタック  
　言語/ライブラリ: Python (Pandas, NumPy, Matplotlib, Seaborn)  
　モデル: XGBoost (不均衡データに対応した重み付け調整)  
　手法: 層別モデリング、PR曲線による閾値最適化、特徴量重要度分析
